## ðŸ”¹ Definition
- Amazon S3 is an **object-based storage service**.
- It provides **unlimited storage capacity**.
- Designed for **high durability, availability, scalability, and security**.
- Commonly used for **backup, archive, static website hosting, big data, and application data**.

---

## ðŸ”¹ Buckets
- Buckets are **logical containers** that store objects.
- Every object must belong to **one bucket**.
- **Bucket Naming Rules**:
  - Bucket names must be **globally unique** across all AWS accounts.
  - Must be **3â€“63 characters long**.
  - Only **lowercase letters, numbers, and hyphens (-)** allowed.
  - Cannot contain underscores or uppercase letters.
  - Cannot be IP-address formatted.
- Bucket is created in a **specific AWS region**.

---

## ðŸ”¹ Objects
- Objects are **actual files** stored in S3.
- Each object consists of:

### Object Components
- **Key**  
  - Unique name of the object within a bucket.
- **Value**  
  - Actual data (bytes of the file).
- **Version ID**  
  - Generated when versioning is enabled.
- **Metadata**  
  - Information about the object (content-type, size, last-modified).
- **Subresources**  
  - ACL (Access Control List)  
  - Torrent  

---

## ðŸ”¹ Data Consistency Model
- **Read-after-write consistency**  
  - For new object PUT operations (immediate access).
- **Eventual consistency**  
  - For overwrite PUTs and DELETE operations.
- Ensures **high performance at global scale**.

## ðŸ”¹ Additional S3 Features

### Versioning
- Maintains multiple versions of an object.
- Protects against accidental deletion or overwrite.
- Works well with lifecycle rules.

---

### Encryption
- **Server-Side Encryption (SSE)**:
  - SSE-S3 (AWS-managed keys)
  - SSE-KMS (KMS-managed keys)
  - SSE-C (Customer-provided keys)
- **Client-Side Encryption**:
  - Encrypt data before uploading to S3.

---

### Replication
- **Cross-Region Replication (CRR)**:
  - Replicate data to another AWS region.
- **Same-Region Replication (SRR)**:
  - Replicate within same region.
- Requires versioning enabled.

---

### Lifecycle Management
- Automates transition between storage classes.
- Can delete objects after defined time.
- Reduces long-term storage cost.

---

### Transfer Acceleration
- Uses **CloudFront edge locations**.
- Speeds up uploads/downloads globally.
- Ideal for long-distance data transfers.

---

### Security & Access Control (Added)
- **Bucket Policies** (JSON-based).
- **IAM Policies**.
- **ACLs** (legacy).
- **Block Public Access** (recommended ON).
- Supports **VPC Endpoint access**.

---

### Static Website Hosting (Added)
- Host static HTML, CSS, JS websites.
- No server required.
- Integrated with CloudFront for CDN.
