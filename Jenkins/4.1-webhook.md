# Jenkins Webhooks: Automating Builds on Code Push

## 1. What is a Webhook?
Currently, our Jenkins pipeline runs when we click **"Build Now"** manually. This is "Polling" (Jenkins asking GitHub: "Do you have new code?").

A **Webhook** works the opposite way.
*   **Sender (GitHub):** "Hey Jenkins, I just received new code from the developer. Here is the payload."
*   **Receiver (Jenkins):** "Thanks for the info. I will start the Pipeline immediately."

This allows for **Continuous Integration**: the moment a developer pushes code, the build starts automatically.

---

## 2. Prerequisites
To set up a Webhook, you need to connect GitHub and Jenkins securely.
1.  **Jenkins Plugin:** Install the **GitHub Plugin** (or GitLab Plugin) in Jenkins.
    *   Go to `Manage Jenkins` -> `Plugins` -> `Available` -> Search `GitHub` -> `Install`.
2.  **Jenkins Pipeline:** Ensure you have a Pipeline (like the Maven one we created in the previous topic) that is linked to your GitHub repository.

---

## 3. Step 1: Configure Jenkins to Listen (The Receiver)
You must tell Jenkins to accept incoming messages from GitHub.

1.  Open your **Pipeline** project configuration in Jenkins.
2.  Scroll down to **Build Triggers**.
3.  Check the box: **GitHub hook trigger for GITScM polling**.
    *   *Note:* Even though it says "polling," this setting allows Jenkins to receive the webhook signal from the GitHub plugin.
4.  Click **Apply** or **Save**.

---

## 4. Step 2: Get the Webhook URL
Jenkins provides a specific URL that you must put into GitHub so they can "shake hands."

1.  Go to **Manage Jenkins** -> **System Configuration**.
2.  Scroll down to the **GitHub** section.
3.  Look for a link or text that says **"GitHub Webhook"** or similar configuration area.
4.  Copy the **URL** displayed there.
    *   It usually looks like: `http://<YOUR-JENKINS-IP>:8080/github-webhook/`
    *   **Important:** This URL ends with a specific suffix (`/github-webhook/`). You cannot just use the main IP address.

---

## 5. Step 3: Configure GitHub (The Sender)
Now, tell GitHub where to send the signal.

1.  Open your **Source Code Repository** on GitHub (where the developer pushes code).
2.  Go to **Settings** (top right menu).
3.  In the left sidebar, click **Webhooks**.
4.  Click **Add webhook**.
5.  **Payload URL:** Paste the URL you copied from Jenkins (e.g., `http://<YOUR-JENKINS-IP>:8080/github-webhook/`).
6.  **Content type:** Select `application/json`.
7.  **Secret:** (Optional but recommended) If Jenkins generated a secret key in the configuration, paste it here. If not, you can leave it blank for testing.
8.  **Events:** Ensure **"Just the push event"** is selected (this is default).
9.  Click **Add webhook**.

---

## 6. Step 4: Test the Automation
Now, simulate what a developer does.

1.  Open your project locally on your machine.
2.  Make a small change (e.g., edit `README.md` or a Java file).
3.  Run the Git commands:
    ```bash
    git add .
    git commit -m "Testing webhook trigger"
    git push origin main
    ```
4.  **Result:**
    *   As soon as you push, GitHub sends a signal to Jenkins.
    *   Jenkins receives the signal.
    *   Jenkins automatically starts the Pipeline build (Pull -> Maven Build -> Package).
    *   You do not need to click "Build Now" in Jenkins anymore.

---

## 7. Common Troubleshooting
*   **Issue:** Webhook is triggered in GitHub logs, but Jenkins doesn't start the build.
    *   **Cause:** The Repository URL defined inside your Jenkins Pipeline script must match the Repository URL where you added the Webhook.
    *   **Fix:** Check your `Jenkinsfile` or Pipeline configuration. Ensure the `git url` matches the GitHub repo hosting the webhook.
*   **Issue:** "Hook not found" or 404 error.
    *   **Cause:** Incorrect URL in GitHub.
    *   **Fix:** Ensure you included the `/github-webhook/` suffix at the end of the Jenkins URL.

## Summary
*   **Install GitHub Plugin** in Jenkins.
*   **Enable Trigger** in Pipeline config ("GitHub hook trigger...").
*   **Copy Webhook URL** from Manage Jenkins -> System Configuration.
*   **Paste URL** into GitHub Repository Settings -> Webhooks.
*   **Push Code** -> Build starts automatically.
